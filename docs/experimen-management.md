# Overview

- 分析において、前処理・モデル選択・ハイパーパラメータチューニングの組み合わせを効率的に管理・実行するために、MLflowを導入する。
- MLOpsを実行するための基盤を構築し、再現性のある実験を実施する。

# Theme

- 電力変圧器の油温（Oil Temperature, OT）を予測するために、ETT-smallデータセット(`ETTh1.csv`, `ETTh2.csv`, `ETTm1.csv`, `ETTm2.csv`)を用いて、時系列データ解析を行う。実験管理フレームワークを活用し、前処理手法やモデル選択、ハイパーパラメータの最適化を体系的に実施する。

# Procedure

1. 試行錯誤用のJupyter Notebookを作成し、データの前処理、モデル選択、ハイパーパラメータチューニングの各ステップを実装する。
    - この際、設定は`exp_<nnn>.yml`ファイルに記述し、各ステップで使用する前処理手法、モデル、ハイパーパラメータを柔軟に変更できるようにする。
2. notebookの内容をもとに、`scripts/main.py`をアップデートし、MLflowを用いて各ステップの実験管理を行う。
    - 前処理ステップでは、欠損値処理、異常値検出、特徴量エンジニアリングを実装する。
    - モデル選択ステップでは、複数のモデル（ARIMA、LSTM、Informerなど）を比較し、最適なモデルを選定する。
    - ハイパーパラメータチューニングステップでは、Grid SearchやRandom Searchを用いて最適なパラメータを探索する。
3. MLflowのUIを用いて、各実験の結果を可視化・比較し、最適な前処理手法、モデル、ハイパーパラメータを特定する。
4. 最終的なモデルを用いてテストデータに対する予測を実行し、結果を評価する。
5. 実験の再現性を確保するために、設定を記述した`exp_<nnn>.yml`をGitHubリポジトリに保存する。

# Naming Rules
- 実験名: `exp_<nnn>`
    - 例: `exp_001`
- 実験用ノートブック: `exp_<nnn>.ipynb`
    - 例: `exp_001.ipynb`
- モデルフォルダ: `model_<nnn>/`
    - 例: `model_001/`
- モデル名: `model_<nnn>`
    - 例: `model_001`
    - モデル詳細は、`exp_<nnn>.yml`に記述することで中央管理する。
- 前処理後データ: `p_<nnn>.npz`
    - 例: `p_001.npz`
- 実験設定ファイル: `exp_<nnn>.yml`
    - 例: `exp_001.yml`
    - 各ステップで使用する前処理手法、モデル、ハイパーパラメータを記述する。
- スクリプト: `main.py`
    - 実験管理のメインスクリプトとして使用する。
    - エントリーポイントとして機能し、各ステップの処理を呼び出す。
